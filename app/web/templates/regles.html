{% endmacro %} {% call sec('s0', '0 · Séquence de clôture (6 étapes)', true) %} Ordre d'exécution, par établissement : 1. Synchronisation des clients → 2. Génération du CSV (caisse + factures, un seul fichier) → 3. Import du CSV dans Pennylane (manuel) → 4. Cadrage de l'import par Vaelan (vérification API) → 5. Attache des justificatifs (PDF factures) → 6. Lettrage. Le tableau de suivi (page d'accueil société) trace « jusqu'à quelle date » et « réalisé le… » par étape. Couleur : vert si couvert jusqu'au dernier mois TERMINÉ (fin du mois précédent), sinon orange « mois à clôturer ». {% endcall %} {% call sec('s1', '1 · Synchronisation des clients') %} Table de correspondance TopOrder ↔ Pennylane. Clé de matching = SIRET TopOrder ↔ « Identifiant client » Pennylane (champ external_reference, où l'on met le SIRET ; PAS le SIREN qui est partagé entre établissements). Cascade de statut stricte : pas de SIRET → SIRET manquant ; SIRET sans jumeau Pennylane → absent de Pennylane ; jumeau sans compte 411 → incohérent ; jumeau + 411 → ok. Une ligne du mapping introuvable dans TopOrder → absent TopOrder. Vaelan détecte ; la correction se fait dans TopOrder/Pennylane puis on resynchronise. Les identifiants assignés (asso « ASDR ») sont gérés. {% endcall %} {% call sec('s2', '2 · Génération du CSV (caisse + factures, fichier unique)') %} Un seul CSV contenant deux journaux (Pennylane les sépare par la colonne « Code journal ») : TOSLT (caisse) et TOSLF (reclassement). Le ticket est la vérité fiscale ; tout le CA va au 70101 (= le Z).
  • Tickets anonymes → 1 écriture agrégée par jour : crédit 70101/taux + TVA collectée ; débit encaissements (CB/Espèce/TR, rendu monnaie netté) ; écart de caisse résiduel.
  • Tickets facturés → 1 écriture par facture : crédit 70101/taux + TVA ; débit 411 client du brut TTC (lettrage F<n°> = la facture) ; par paiement : débit encaissement + crédit 411 (règlement, lettré). Solde 411 restant = impayé.
Format CSV validé import Pennylane : comptes par NUMÉRO, journal par CODE (TOSLT), taux en virgule (2,1%), catégorie analytique sur TOUTES les lignes (Etablissement / nom court). Libellé de pièce « SM280526 #SMT03 » (jour en tête, lot en queue). Identifiant de lot supprimable d'un bloc. Noms de fichiers préfixés AAAAMMJJ HHMM (heure Réunion). {% endcall %} {% call sec('s2b', '2b · Pré-vol & cadrage (verrou de génération)') %} Pré-vol client (souple) : la génération est refusée uniquement si une facture de la période ne se route vers aucun compte 411 ; on liste les clients à corriger.
Cadrage : on compare le CA recalculé sur les tickets à la synthèse PDF. Le CA Total (et la période) sont BLOQUANTS. Un écart par mode de paiement (ex. CB) vient des tickets facturés payés comptant que la synthèse TopOrder classe hors « caisse ». Vaelan les RÉCONCILIE : il vérifie que l'écart est entièrement couvert par les paiements de factures encaissés en caisse (bookés au crédit du 411, lettrés F<n°>). Si oui → contrôle « COUVERT » (l'écart ne vient pas de nulle part, il est traité dans le CSV). Si l'écart dépasse les paiements de factures → avertissement « non expliqué ». Tolérance ± 0,05 €. Compte rendu PDF : Synthèse | API tickets | CSV agrégé par poste + section « Réconciliation des écarts de paiement ». {% endcall %} {% call sec('s4', '4 · Cadrage de l\'import par Vaelan (vérification Pennylane)') %} Re-jouable à tout moment. Vaelan relit les écritures TOSLT du période dans Pennylane (via l'API), les agrège par compte, et compare à l'attendu = les CSV générés et stockés. Détecte toute suppression / modification / ajout survenu depuis l'import. Horodate la dernière vérification (« c'était bon à cette date »). Compte rendu PDF par compte (Attendu | Pennylane | OK/Écart). {% endcall %} {% call sec('s2c', 'Factures (TOSLF) & règlements — dans le même CSV') %} Reclassement (journal TOSLF, par facture) : débit 70101 / crédit 7012 (B2B) ou 70102 (B2C), HT par taux (avoirs inversés) → le CA facturé sort du 70101, net nul sur le CA total, pas de double comptage.
Écriture facture (journal TOSLT) : CA 70101/TVA + débit 411 brut (la facture) + paiements comptant (encaissement + crédit 411), lettrés F<n°>. Solde 411 = impayé.
Paiements décalés : une facture peut être réglée en caisse plus tard. Deux flux par période — CA par date de ticket ; règlements par date de paiement (liaison facture↔paiement via orderPaymentId), débit encaissement / crédit 411 F<n°>, lettrés à travers les périodes. {% endcall %} {% call sec('s8', '5-6 · Justificatifs & lettrage — à venir') %} Étapes distinctes par type de justificatif (ex. « Attacher les PDF factures aux écritures factures »), chacune finissant par un rapport de complétude (« toutes attachées, 0 écriture sans PDF »). Lettrage : soldes 411 par n° de facture, rapprochés avec les règlements (caisse, virements). {% endcall %} {% call sec('cfg', 'Configuration & traçabilité') %} Tous les comptes (70101/70102/7012, TVA par taux, encaissements par établissement), journaux (code + id) et catégories analytiques sont éditables (page Configuration) ; seules les valeurs modifiées sont stockées. Chaque tâche conserve : la synthèse fournie, le CSV généré, le compte rendu PDF, la version de Vaelan et l'utilisateur. Tâches relançables ; suivi réinitialisable par établissement. {% endcall %}
